# PRD Command

Invoke the **prd-creator** agent to create a Product Requirements Document for $ARGUMENTS.

## Output Structure

1. **Problem Statement** — What problem, who has it, current solutions
2. **User Stories** — As a [user], I want [action], so that [benefit]
3. **Functional Requirements** — Must-have, should-have, could-have
4. **Non-Functional Requirements** — Performance, security, accessibility
5. **Technical Requirements** — Architecture, APIs, data models
6. **Success Metrics** — Primary KPIs, how/when to measure
7. **Timeline** — MVP scope, Phase 2, future considerations
8. **Risks** — Technical, adoption, competitive

Every user story must have acceptance criteria.
Requirements must be specific and testable.
