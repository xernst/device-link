# Content Engine Skill

## Content Types

### Technical Documentation
- Architecture overviews
- API documentation
- Setup guides and tutorials
- Migration guides

### Communication
- Meeting agendas and notes
- Status reports
- Stakeholder updates
- Design proposals

### Research Deliverables
- Competitive analysis reports
- Technology evaluation documents
- Market research briefs
- Due diligence reports

## Writing Principles

1. **Lead with the conclusion** — Don't bury the lede
2. **Be specific** — Use numbers, dates, names
3. **Use structure** — Headers, lists, tables for scannability
4. **Include evidence** — Sources, data, examples
5. **Know your audience** — Technical vs. business language

## Content Quality Checklist

- [ ] Clear purpose stated upfront
- [ ] Structured with headers and sections
- [ ] Evidence supports claims
- [ ] Action items are specific
- [ ] Audience-appropriate language
- [ ] Proofread for accuracy
